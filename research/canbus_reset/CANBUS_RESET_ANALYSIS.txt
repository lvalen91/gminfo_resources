PROBABLE RESET SEQUENCE
-----------------------
Based on analysis, the likely UDS message sequence is:

1. DiagnosticSessionControl - Enter Extended Session
   Request:  0x10 0x03
   Response: 0x50 0x03 ...

2. SecurityAccess - Request Seed
   Request:  0x27 0x01
   Response: 0x67 0x01 [seed bytes]

3. SecurityAccess - Send Key
   Request:  0x27 0x02 [calculated key]
   Response: 0x67 0x02

4. RoutineControl - Start Reset Routine
   Request:  0x31 0x01 0x32 0x08 [optional params]
   Response: 0x71 0x01 0x32 0x08 ...

5. ECUReset - Perform Reset
   Request:  0x11 0x01 (or 0x11 0x03)
   Response: 0x51 0x01 (or 0x51 0x03)

CAN ADDRESSING
--------------
The CAN arbitration IDs are NOT hardcoded in this DLL. They are managed by
the tisvcsv4.dll VCS layer based on:
- Vehicle configuration (from TIS2Web/SPS)
- Target module (A11 Radio)
- Protocol selection (HS-CAN, GMLAN, etc.)

For GM vehicles, typical A11 Radio addressing:
- GMLAN: May use physical addressing via Serial Gateway
- Direct HS-CAN: Likely 0x641 (Tx) / 0x249 (Rx) or similar

The Type4 Application abstracts these details - SPS handles the routing.

SERIAL GATEWAY VS DIRECT

CONCLUSION
----------
This Type4 Application provides a CAN-level factory data reset for the A11
Radio module. It requires:
- GM SPS/TIS2Web infrastructure
- Proper VCI connection
- Security Access authentication
- Vehicle in Run/Service power mode

The reset uses RoutineControl (0x31) with routine ID 0x3208, followed by
ECUReset (0x11). The security algorithm and actual CAN addressing are
managed by the VCS layer and vehicle configuration data from GM servers.

For manual execution outside SPS, you would need:
1. The Security Access seed-key algorithm
2. Proper CAN arbitration IDs for your specific vehicle
3. UDS-capable diagnostic tool
